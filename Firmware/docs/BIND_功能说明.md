# SiK 一键对频（Bind）功能说明

## 概述

对频功能允许两台 SiK 数传模块在 **NETID（网络 ID）不一致** 时，通过按一下 PAIR 按键，将主机的全套参数（NETID、空口速率、频率范围等）自动同步到从机，完成配对。

对频完成后两台模块自动软件复位，重启后即可正常通信，**无需手动断电重启**。

---

## 默认参数说明

以下为本固件的出厂默认参数，两台模块出厂时 **NETID 相同（100）**，可直接上电建链。  
若需对频，只需将从机 NETID 改为其他值，再执行对频操作即可。

### 主机默认参数（ATS16=1）

```
S0:FORMAT=27
S1:SERIAL_SPEED=57      # 串口波特率 57600
S2:AIR_SPEED=64         # 空口速率 64kbps
S3:NETID=100            # 网络 ID，两端必须一致才能通信
S4:TXPOWER=20           # 发射功率 20dBm（最大）
S5:ECC=0                # 关闭 Golay 纠错（0=关，1=开）
S6:MAVLINK=1            # MAVLink 帧格式（0=忽略，1=使用）
S7:OPPRESEND=0          # 机会重传（已禁用）
S8:MIN_FREQ=433050      # 跳频最低频率 433.050 MHz
S9:MAX_FREQ=434790      # 跳频最高频率 434.790 MHz
S10:NUM_CHANNELS=10     # 跳频信道数
S11:DUTY_CYCLE=100      # 占空比 100%
S12:LBT_RSSI=0          # 先听后说门限（0=禁用）
S13:MANCHESTER=0        # Manchester 编码（0=关）
S14:RTSCTS=0            # 硬件流控（0=关）
S15:MAX_WINDOW=131      # 最大发送窗口（ms）
S16:BIND_ROLE=1         # 对频角色：1=主机
```

### 从机默认参数（ATS16=0）

```
S0:FORMAT=27
S1:SERIAL_SPEED=57
S2:AIR_SPEED=64
S3:NETID=100            # 与主机相同才能建链
S4:TXPOWER=20
S5:ECC=0
S6:MAVLINK=1
S7:OPPRESEND=0
S8:MIN_FREQ=433050
S9:MAX_FREQ=434790
S10:NUM_CHANNELS=10
S11:DUTY_CYCLE=100
S12:LBT_RSSI=0
S13:MANCHESTER=0
S14:RTSCTS=0
S15:MAX_WINDOW=131
S16:BIND_ROLE=0         # 对频角色：0=从机
```

> ⚠️ **关键说明**
> - `S3:NETID` 两端必须完全一致，否则射频硬件层直接过滤掉对方的包，**无法建链**。
> - `S16:BIND_ROLE` 决定对频角色，对频完成后从机的该参数**不会被主机覆盖**，永久保持 0。
> - 其余参数（S2、S4～S15）由主机通过对频同步到从机，无需手动在从机上逐一配置。

---

## 主从机角色配置

对频分为**主机**和**从机**两个角色，通过参数 **S16（BIND_ROLE）** 区分：

| 参数值 | 角色 | 行为 |
|--------|------|------|
| `ATS16=1` | **主机（Master）** | 按键后广播本机完整参数表 |
| `ATS16=0` | **从机（Slave）** | 按键后监听，收到主机参数后写入并重启 |

### 查看当前配置

通过串口助手进入 AT 指令模式（发送 `+++`，收到 `OK`），然后：

```
ATI5
```

输出示例（从机）：

```
S0:FORMAT=27 S1:SERIAL_SPEED=57 S2:AIR_SPEED=64 S3:NETID=100
S4:TXPOWER=20 S5:ECC=0 S6:MAVLINK=1 ... S16:BIND_ROLE=0
```

### 设置主从机角色

**主机**（固定一台，通常是地面站端）：

```
ATS16=1
AT&W
ATZ
```

**从机**（固定另一台，通常是飞机/设备端）：

```
ATS16=0
AT&W
ATZ
```

> **注意**：`AT&W` 将参数写入 Flash，`ATZ` 触发软件重启使参数生效。  
> `S16` 参数在对频成功后**不会**被覆盖，主从角色永久固定。

## 功能测试（首次验证）

以下步骤适合首次验证对频功能是否正常工作。

### 第一步：确认两台模块当前参数

分别用串口助手连接两台模块（波特率 57600），进入 AT 模式：

```
+++
```

收到 `OK` 后，查看参数：

```
ATI5
```

记录两台的 `S3:NETID` 和 `S16:BIND_ROLE`，确认当前状态。

---

### 第二步：设置主从机角色

**主机**（串口连接主机，执行）：

```
ATS16=1
AT&W
ATZ
```

**从机**（串口连接从机，执行）：

```
ATS16=0
AT&W
ATZ
```

> `AT&W` 写入 Flash，`ATZ` 重启使参数生效。重启后串口会断开，重新连接即可。

---

### 第三步：故意制造 NETID 不一致

在**从机**上将 NETID 改为一个与主机不同的值（例如主机是 100，从机改成 99）：

```
ATS3=99
AT&W
ATZ
```

此时从机 NETID=99，主机 NETID=100，双方无法正常通信（红灯慢闪）。

> **说明**：对频的目的正是解决 NETID 不一致问题，所以需要先制造不一致再测试。

---

### 第四步：执行对频

两台模块退出 AT 模式（等待 1 秒不输入，或发送 `ATO`），然后：

1. **先按从机 PAIR 键**（或发 `ATB`）→ 从机 LED 红绿交替快闪
2. **再按主机 PAIR 键**（或发 `ATB`）→ 主机 LED 红绿交替快闪

两台均在快闪说明双方已进入对频模式。

---

### 第五步：等待自动完成

约 **5~10 秒**后观察 LED：

| 观察到的现象 | 说明 |
|------------|------|
| 从机绿灯常亮 1 秒后重启 | 从机收到主机参数，正在写入 Flash |
| 主机绿灯常亮 1 秒后重启 | 主机发包完成，自动重启同步 |
| 双方重启后绿灯常亮 | ✅ 对频成功，链路已建立 |

---

### 第六步：验证结果

重启后，在任意一台上进入 AT 模式，查看 NETID：

```
+++
ATI5
```

**预期结果**：

- 从机 `S3:NETID` 已变为主机的值（例如 25）
- 从机 `S16:BIND_ROLE` 仍为 0（未被覆盖）
- 两台 NETID 一致，绿灯常亮，通信正常

---

## 对频操作步骤

### 前提条件

- 两台模块均已烧录支持对频功能的固件（`radio~hm_trp.ihx`）
- 主机已设置 `ATS16=1`，从机已设置 `ATS16=0`
- 两台模块上电，正常工作（LED 状态正常）

### 操作流程

> ⚠️ **顺序很重要：必须先按从机，再按主机**  
> 主机进入对频后只广播约 5 包（~1 秒）就自动重启；从机进入后最长监听 30 秒。  
> 若先按主机，主机广播完毕后从机还未进入监听，将会错过，导致对频失败。

```
① 先按从机 PAIR 键
         │
         ▼
   从机进入监听模式
  （LED 红绿交替快闪，最长等待 30 秒）

② 再按主机 PAIR 键（30 秒内）
         │
         ▼
   主机进入广播模式
  （LED 红绿交替快闪，广播 5 包 ≈ 1 秒）
         │
         ▼
   从机收到参数包 → 写入 Flash → 绿灯 1 秒 → 自动重启
   主机发完 5 包  → 绿灯 1 秒 → 自动重启
         │
         ▼
   双方以新 NETID 重新建链，连接成功（绿灯常亮）
```

### LED 指示说明

| LED 状态 | 含义 |
|----------|------|
| 红绿交替快闪 | 正在对频（主机或从机均如此） |
| 绿灯常亮 1 秒 | 对频成功，即将重启 |
| 绿灯常亮（重启后） | 链路已建立，正常通信 |
| 红灯慢闪（重启后） | 未建链，等待对端 |

---

## 通过 AT 指令触发对频

除按键外，也可通过串口 AT 指令进入对频模式（等效按键）：

```
+++
OK
ATB
```

适合无法方便操作按键的场合（如设备已装机）。

## 对频完成后的参数同步规则

| 参数 | 是否从主机同步到从机 |
|------|---------------------|
| S1: SERIAL_SPEED | ✅ 同步 |
| S2: AIR_SPEED | ✅ 同步 |
| S3: NETID | ✅ 同步（核心目的） |
| S4: TXPOWER | ✅ 同步 |
| S5: ECC | ✅ 同步 |
| S8: MIN_FREQ | ✅ 同步 |
| S9: MAX_FREQ | ✅ 同步 |
| S10: NUM_CHANNELS | ✅ 同步 |
| **S16: BIND_ROLE** | ❌ **不同步**（从机保留自己的角色 0） |

## 典型应用场景

### 场景：新模块入网

1. 新从机出厂 `NETID` 可能与现有主机不同
2. 主机保持上电运行
3. **先按从机 PAIR**，再在 30 秒内**按主机 PAIR**
4. 约 5~10 秒后双方自动重启，链路建立完成

### 场景：更换网络参数

1. 在主机上通过 AT 指令修改 NETID（`ATS3=新值`，`AT&W`，`ATZ` 重启）
2. 主机重启后重新按 PAIR，从机也按 PAIR
3. 完成同步，双方以新 NETID 通信

## 硬件说明（HM-TRP 定制板）

PAIR 按键连接在 **Si1000 的 P1.7 引脚**，低电平有效。  
按键电路：`GND → [PAIR 按键] → R6(10kΩ) → VDD_MCU`（P1.7 通过 R6 上拉）。

> 若硬件设计中 PAIR 按键通过电容 AC 耦合（如串联 100nF 隔直电容），固件采用下降沿触发，**按一下即触发，无需长按**。
