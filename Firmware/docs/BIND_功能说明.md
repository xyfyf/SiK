# SiK 一键对频（Bind）功能说明

## 概述

对频功能允许两台 SiK 数传模块在 **NETID（网络 ID）不一致** 时，通过各按一下 PAIR 按键，自动同步参数完成配对。

**【对称对频】：无需区分主从顺序，先按哪台都可以。**  
两台模块按键后均先进入约 1 秒的监听等待，随后未收到对端包的那台自动切换为发送角色，广播本机参数；另一台收到参数后写入 Flash 并自动重启。整个过程无需手动断电重启，约 5~10 秒自动完成。

---

## 默认参数说明

以下为本固件的出厂默认参数，两台模块出厂时 **NETID 相同（100）**，可直接上电建链。  
若需对频，只需将从机 NETID 改为其他值，再执行对频操作即可。

### 主机默认参数（ATS16=1）

```
S0:FORMAT=27
S1:SERIAL_SPEED=57      # 串口波特率 57600
S2:AIR_SPEED=64         # 空口速率 64kbps
S3:NETID=100            # 网络 ID，两端必须一致才能通信
S4:TXPOWER=20           # 发射功率 20dBm（最大）
S5:ECC=0                # 关闭 Golay 纠错（0=关，1=开）
S6:MAVLINK=1            # MAVLink 帧格式（0=忽略，1=使用）
S7:OPPRESEND=0          # 机会重传（已禁用）
S8:MIN_FREQ=433050      # 跳频最低频率 433.050 MHz
S9:MAX_FREQ=434790      # 跳频最高频率 434.790 MHz
S10:NUM_CHANNELS=10     # 跳频信道数
S11:DUTY_CYCLE=100      # 占空比 100%
S12:LBT_RSSI=0          # 先听后说门限（0=禁用）
S13:MANCHESTER=0        # Manchester 编码（0=关）
S14:RTSCTS=0            # 硬件流控（0=关）
S15:MAX_WINDOW=131      # 最大发送窗口（ms）
S16:BIND_ROLE=1         # 对频角色：1=主机
```

### 从机默认参数（ATS16=0）

```
S0:FORMAT=27
S1:SERIAL_SPEED=57
S2:AIR_SPEED=64
S3:NETID=100            # 与主机相同才能建链
S4:TXPOWER=20
S5:ECC=0
S6:MAVLINK=1
S7:OPPRESEND=0
S8:MIN_FREQ=433050
S9:MAX_FREQ=434790
S10:NUM_CHANNELS=10
S11:DUTY_CYCLE=100
S12:LBT_RSSI=0
S13:MANCHESTER=0
S14:RTSCTS=0
S15:MAX_WINDOW=131
S16:BIND_ROLE=0         # 对频角色：0=从机
```

> ⚠️ **关键说明**
> - `S3:NETID` 两端必须完全一致，否则射频硬件层直接过滤掉对方的包，**无法建链**。
> - `S16:BIND_ROLE` 在新版固件中**不再影响对频顺序**，保留在参数表中仅作历史兼容用途。对频完成后该参数**不会被对端覆盖**，各自保留原值。
> - 其余参数（S2、S4～S15）由发送方通过对频同步到接收方，无需手动在另一台上逐一配置。

---

## S16 参数说明

`S16:BIND_ROLE` 是历史遗留参数，**新版固件已不再使用它决定对频顺序**。  
可以忽略该参数，或保留原有设置，对频行为完全不受影响。

### 查看当前配置

通过串口助手进入 AT 指令模式（发送 `+++`，收到 `OK`），然后：

```
ATI5
```

输出示例：

```
S0:FORMAT=27 S1:SERIAL_SPEED=57 S2:AIR_SPEED=64 S3:NETID=100
S4:TXPOWER=20 S5:ECC=0 S6:MAVLINK=1 ... S16:BIND_ROLE=0
```

> `S16` 在对频过程中不起作用，对频完成后也**不会被对端覆盖**，各自保留原值。

## 功能测试（首次验证）

以下步骤适合首次验证对频功能是否正常工作。

### 第一步：确认两台模块当前参数

分别用串口助手连接两台模块（波特率 57600），进入 AT 模式：

```
+++
```

收到 `OK` 后，查看参数：

```
ATI5
```

记录两台的 `S3:NETID` 和 `S16:BIND_ROLE`，确认当前状态。

---

### 第二步：故意制造 NETID 不一致

在**其中一台**上将 NETID 改为一个与另一台不同的值（例如 A 是 100，B 改成 99）：

```
ATS3=99
AT&W
ATZ
```

此时 B 的 NETID=99，A 的 NETID=100，双方无法正常通信（红灯慢闪）。

> **说明**：对频的目的正是解决 NETID 不一致问题，所以需要先制造不一致再测试。

---

### 第三步：执行对频

两台模块退出 AT 模式（等待 1 秒不输入，或发送 `ATO`），然后：

**任意顺序分别按两台的 PAIR 键**（或发 `ATB`），无需在意谁先谁后。

两台均出现 LED 红绿交替快闪，说明已进入对频模式。

> 若只有一台在闪，说明另一台未收到按键信号，重新按一下即可。

---

### 第四步：等待自动完成

约 **5~10 秒**后观察 LED：

| 观察到的现象 | 说明 |
|------------|------|
| 其中一台绿灯常亮 1 秒后重启 | 接收方收到参数，正在写入 Flash |
| 另一台绿灯常亮 1 秒后重启 | 发送方发包完成，自动重启同步 |
| 双方重启后绿灯常亮 | ✅ 对频成功，链路已建立 |

---

### 第五步：验证结果

重启后，在任意一台上进入 AT 模式，查看 NETID：

```
+++
ATI5
```

**预期结果**：

- 两台 `S3:NETID` 已相同（均为发送方的值）
- `S16:BIND_ROLE` 各自保留原值（未被对端覆盖）
- 绿灯常亮，通信正常

---

## 对频操作步骤

### 前提条件

- 两台模块均已烧录支持对频功能的固件（`radio~hm_trp.ihx`）
- 两台模块上电，正常工作（LED 状态正常）

### 操作流程

> ✅ **无需区分顺序：两台模块分别按一下 PAIR 键即可，先后顺序不限。**

```
① 任意一台按下 PAIR 键（记为设备 A）
         │
         ▼
   设备 A 进入"探测监听"阶段（约 1 秒）
  （LED 红绿交替快闪）

② 另一台按下 PAIR 键（记为设备 B，在 30 秒内）
         │
         ▼
   设备 B 进入"探测监听"阶段（约 1 秒）
  （LED 红绿交替快闪）

── 探测监听到期后（先到期的那台自动切换为发送方）──

   发送方：广播本机参数（5 包）→ 绿灯 1 秒 → 自动重启
   接收方：收到参数包 → 写入 Flash → 绿灯 1 秒 → 自动重启
         │
         ▼
   双方以相同 NETID 重新建链，连接成功（绿灯常亮）
```

> **同时按键的情况**：两台设备的探测监听时长有约 0~262ms 的随机差异，先结束的自动变为发送方，另一台收包后完成对频，完全自动处理，无需干预。

### LED 指示说明

| LED 状态 | 含义 |
|----------|------|
| 红绿交替快闪 | 正在对频（主机或从机均如此） |
| 绿灯常亮 1 秒 | 对频成功，即将重启 |
| 绿灯常亮（重启后） | 链路已建立，正常通信 |
| 红灯慢闪（重启后） | 未建链，等待对端 |

---

## 通过 AT 指令触发对频

除按键外，也可通过串口 AT 指令进入对频模式（等效按键）：

```
+++
OK
ATB
```

适合无法方便操作按键的场合（如设备已装机）。

## 对频完成后的参数同步规则

| 参数 | 是否从主机同步到从机 |
|------|---------------------|
| S1: SERIAL_SPEED | ✅ 同步 |
| S2: AIR_SPEED | ✅ 同步 |
| S3: NETID | ✅ 同步（核心目的） |
| S4: TXPOWER | ✅ 同步 |
| S5: ECC | ✅ 同步 |
| S8: MIN_FREQ | ✅ 同步 |
| S9: MAX_FREQ | ✅ 同步 |
| S10: NUM_CHANNELS | ✅ 同步 |
| **S16: BIND_ROLE** | ❌ **不同步**（各自保留原值，对频顺序已不依赖此参数） |

## 典型应用场景

### 场景：新模块入网

1. 新模块出厂 `NETID` 可能与现有模块不同
2. 两台模块上电
3. **分别按下各自的 PAIR 键**（顺序不限，30 秒内完成即可）
4. 约 5~10 秒后双方自动重启，链路建立完成

### 场景：更换网络参数

1. 在其中一台上通过 AT 指令修改 NETID（`ATS3=新值`，`AT&W`，`ATZ` 重启）
2. 重启后，两台分别按 PAIR 键（顺序不限）
3. 完成同步，双方以新 NETID 通信

## 硬件说明（HM-TRP 定制板）

PAIR 按键连接在 **Si1000 的 P1.7 引脚**，低电平有效。  
按键电路：`GND → [PAIR 按键] → R6(10kΩ) → VDD_MCU`（P1.7 通过 R6 上拉）。

> 若硬件设计中 PAIR 按键通过电容 AC 耦合（如串联 100nF 隔直电容），固件采用下降沿触发，**按一下即触发，无需长按**。
